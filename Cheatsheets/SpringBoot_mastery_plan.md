# ✅ React Mastery Checklist

## 1. Débutant – "Je connais React"
- [ ] **Création projet (CRA/Vite)**  
  → Lancer une app avec `create-react-app` ou Vite.
- [ ] **JSX & Components**  
  → Créer des composants fonctionnels et les utiliser.
- [ ] **Props & State**  
  → Passer des données via props, gérer l’état avec `useState`.
- [ ] **Events & Forms**  
  → Gérer les événements (`onClick`, `onChange`), formulaires contrôlés.

## 2. Intermédiaire – "Je suis à l’aise"
- [ ] **Hooks**  
  → Utiliser `useEffect`, `useContext`, `useReducer`.
- [ ] **Routing**  
  → Utiliser React Router (`BrowserRouter`, `Route`, `Link`).
- [ ] **HTTP Requests**  
  → Consommer une API avec `fetch` ou Axios.
- [ ] **Styling**  
  → Utiliser CSS Modules, Styled Components ou Tailwind CSS.
- [ ] **Context API**  
  → Partager du state global sans props drilling.

## 3. Avancé – "Je maîtrise React"
- [ ] **Performance**  
  → Utiliser `React.memo`, `useCallback`, `useMemo` pour éviter les rerenders.
- [ ] **Custom Hooks**  
  → Écrire ses propres hooks pour réutiliser de la logique.
- [ ] **Forms avancés**  
  → Utiliser Formik/React Hook Form pour validation.
- [ ] **Testing**  
  → Écrire des tests unitaires avec Jest/React Testing Library.
- [ ] **SSR**  
  → Comprendre Next.js pour rendu côté serveur.

## 4. Expert – "Je domine React"
- [ ] **State Management avancé**  
  → Maîtriser Redux Toolkit, Zustand, Recoil.
- [ ] **React Concurrent & Suspense**  
  → Comprendre le concurrent mode et le data fetching moderne.
- [ ] **Architecture d’app à grande échelle**  
  → Structurer un projet React en Clean Architecture.
- [ ] **Optimisation bundle**  
  → Code splitting, dynamic imports, lazy loading.
- [ ] **Contributions**  
  → Créer une lib React, contribuer à open-source (Next.js, React).
