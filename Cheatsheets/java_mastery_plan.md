# ✅ Java Mastery Checklist

## 1. Débutant – "Je connais Java"

- [ ] **Syntaxe de base**  
  → Être capable d’écrire des classes avec un `main`, utiliser les types primitifs (`int`, `double`, `boolean`), les opérateurs et les structures de contrôle (`if`, `for`, `while`).

- [ ] **Variables et portée**  
  → Comprendre la différence entre variables locales, de classe (`static`) et d’instance.

- [ ] **POO basique**  
  → Savoir définir une classe, instancier des objets avec `new`, utiliser getters/setters.

- [ ] **Chaînes et tableaux**  
  → Manipuler `String`, `StringBuilder` et travailler avec des tableaux (`int[]`).

- [ ] **Compilation et exécution**  
  → Savoir compiler avec `javac` et exécuter avec `java`.

---

## 2. Intermédiaire – "Je suis à l’aise avec Java"

- [ ] **Principes POO (encapsulation, héritage, polymorphisme, abstraction)**  
  → Savoir appliquer ces principes dans des classes concrètes.

- [ ] **Interfaces et classes abstraites**  
  → Comprendre quand utiliser une interface (`implements`) vs une classe abstraite (`extends`).

- [ ] **Collections (List, Set, Map, Queue)**  
  → Savoir choisir la bonne collection (ArrayList vs LinkedList, HashMap vs TreeMap).

- [ ] **Generics**  
  → Utiliser `List<String>`, `Map<Integer, User>`, créer des classes génériques simples.

- [ ] **Gestion des exceptions**  
  → Savoir utiliser `try/catch/finally`, déclarer/propager (`throws`), différencier checked vs unchecked exceptions.

- [ ] **I/O de base (fichiers, flux)**  
  → Lire/écrire un fichier texte avec `FileReader`, `BufferedReader`, `FileWriter`.

- [ ] **Packages et organisation de projet**  
  → Structurer son code avec des packages (`com.project.utils`), comprendre le classpath.

- [ ] **Outils de build (Maven/Gradle)**  
  → Créer un projet avec Maven, comprendre `pom.xml`, gérer des dépendances externes.

---

## 3. Avancé – "Je maîtrise Java"

- [ ] **API Collections avancées & Streams**  
  → Utiliser `stream().map().filter().collect()` et comprendre les avantages fonctionnels.

- [ ] **Programmation fonctionnelle (lambdas)**  
  → Utiliser `() -> {}` pour définir des fonctions anonymes, trier avec lambdas (`Comparator.comparing()`).

- [ ] **Threads et concurrence**  
  → Créer des threads (`Thread`, `Runnable`), utiliser `ExecutorService`, synchroniser (`synchronized`), futures et `CompletableFuture`.

- [ ] **Réflexion (Reflection API)**  
  → Charger des classes dynamiquement (`Class.forName()`), manipuler méthodes et attributs.

- [ ] **Annotations**  
  → Créer et utiliser des annotations personnalisées, comprendre leur usage avec frameworks (Spring, Hibernate).

- [ ] **Java Time API (java.time)**  
  → Utiliser `LocalDate`, `LocalDateTime`, `Instant`, les formatter (`DateTimeFormatter`).

- [ ] **Records et Sealed classes (Java 17+)**  
  → Créer des `record` pour des DTOs immuables, restreindre une hiérarchie avec `sealed`.

- [ ] **Tests unitaires (JUnit/Mockito)**  
  → Écrire des tests JUnit, utiliser Mockito pour simuler des dépendances.

- [ ] **Design Patterns**  
  → Implémenter Singleton, Factory, Builder, Observer et comprendre leurs usages.

- [ ] **Modules (Java 9+)**  
  → Comprendre `module-info.java`, créer des projets modulaires.

---

## 4. Expert – "Je domine Java"

- [ ] **JVM & mémoire**  
  → Comprendre heap, stack, garbage collector, JIT compiler, tuning mémoire (`-Xmx`, `-Xms`).

- [ ] **Classloaders**  
  → Comprendre la hiérarchie des classloaders et résoudre les conflits de dépendances.

- [ ] **Performance et profiling**  
  → Utiliser VisualVM, JFR ou Mission Control pour analyser mémoire, threads et performances.

- [ ] **Sécurité en Java**  
  → Comprendre le `SecurityManager` (déprécié mais historique), manipuler cryptographie (`javax.crypto`).

- [ ] **Réseau (Java Networking)**  
  → Utiliser `Socket`, `ServerSocket`, `HttpClient` moderne (Java 11+).

- [ ] **Frameworks majeurs**  
  → Maîtriser Spring / Spring Boot pour les applications backend, Hibernate pour la persistance, Jakarta EE pour les APIs standard.

- [ ] **Clean code & architecture**  
  → Appliquer SOLID, TDD, Clean Architecture, Domain-Driven Design.

- [ ] **Applications industrielles**  
  → Être capable de concevoir et maintenir de larges systèmes Java (microservices, APIs REST, projets multi-modules).
